Section I: Bezier Curves and Surfaces

Part 1: Bezier curves with 1D de Casteljau subdivision

De Casteljau's algorithm is a recursive method for evaluating a point on a Bezier curve. Given n control points and a parameter t (between 0 and 1), the algorithm repeatedly performs linear interpolation between adjacent points, reducing the count by one each step until a single point remains — the point on the curve at parameter t.

Each interpolated point is computed as: \[ \text{newPoint} = (1 - t) \cdot \text{point}[i] + t \cdot \text{point}[i+1] \]

In my implementation, evaluateStep() performs one level of this reduction: given n points, it returns n-1 interpolated points. To trace the full curve, this function is called repeatedly at many values of t.

Part 2: Bezier surfaces with separable 1D de Casteljau

De Casteljau's algorithm extends to Bezier surfaces by applying it twice — once for each dimension. Given an n×n grid of control points and parameters u and v, the algorithm first runs on each row using parameter u, reducing each row down to a single point. This produces a column of n points, which is then evaluated using parameter v to yield the final surface point.

This was implemented across three functions. evaluateStep() performs one level of interpolation on a vector of 3D points (identical in structure to Part 1 but operating in 3D). evaluate1D() repeatedly calls evaluateStep() until only one point remains, fully collapsing a row or column. evaluate() ties these together by calling evaluate1D() on each row with parameter u, then calling evaluate1D() on the resulting column of points with parameter v.

Section II: Triangle Meshes and Half-Edge Data Structure

Part 3: Area-weighted vertex normals

To compute area-weighted vertex normals, I iterated through all triangles surrounding a given vertex using the half-edge data structure. Starting from the vertex's halfedge, I rotated around the vertex by repeatedly calling twin()->next(), visiting each neighboring triangle.

For each triangle, I then retrieved the positions of its three vertices and computed two edge vectors (p1 - p0) and (p2 - p0), then took their cross product. This produced a vector perpendicular to the triangle whose magnitude equaled twice the triangle's area — providing area-weighting automatically without any extra computation. I accumulated these cross products into a running sum, then normalized with .unit() at the end to return a unit-length normal vector.

Part 6: Loop subdivision for mesh upsampling

Loop subdivision increases mesh resolution by splitting every edge and repositioning vertices according to weighted averages of their neighbors. My implementation follows five steps:

First, I computed updated positions for all of the existing vertices using the Loop subdivision rule and stored them in vertex->newPosition. For each old vertex with degree n, I computed \( u = 3/16 \) if \( n = 3 \), otherwise \( u = 3/(8n) \), then set: \[ \text{newPosition} = (1 - n \cdot u) \cdot \text{originalPosition} + u \cdot \text{neighborSum} \]

Second, I computed the position each new edge-midpoint vertex took and stored it in edge->newPosition using: \[ \text{newPosition} = \frac{3}{8}(A + B) + \frac{1}{8}(C + D) \] where A and B are the edge endpoints and C and D are the opposite vertices of the two adjacent triangles.

Third, I split every original edge in the mesh. I recorded the number of original edges before the splitting began and used a counter to ensure I only split those edges. This prevented infinite loops from newly created edges being processed. After each split, I copied the precomputed edge->newPosition into the new vertex's newPosition.

Fourth, I flipped any new edge that connected one old vertex and one new vertex. This re-triangulated the mesh into the correct Loop subdivision connectivity pattern.

Fifth, I copied every vertex's newPosition into its final position.

Sharp corners and edges

After repeated subdivision, sharp corners and edges become rounded and smoothed out. This happens because the Loop subdivision rule repositions each vertex as a weighted average of its neighbors — corners with few neighbors get pulled toward the mesh interior, losing their sharpness. Ultimately, this leads to the cube's corners becoming rounder until the mesh begins to resemble a sphere.

Pre-splitting to reduce asymmetry

The cube subdivides asymmetrically because each face has only a single diagonal edge, making the triangulation asymmetric — one corner of each face is connected differently than the others. When Loop subdivision is applied, this asymmetry gets amplified with each pass.

To fix this, I pre-split the diagonal edge on each face of the cube before subdividing. This makes the mesh symmetric along all three axes before any subdivision occurs. When Loop subdivision is then applied, the result is a much more symmetric and rounder shape because each vertex now has a symmetric neighborhood of neighbors to average over.
